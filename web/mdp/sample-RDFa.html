Creative Commons Example copyright statements using RDFa

The following version of the statement is derived from the example given at the CC web site using their wizard. It includes two links to the CC web site for this license type (cc-sa, Share-Alike) -- one on the image and one on the text.

My Life as a Dog Phillip Farber by Phillip Farber - This work is licensed under a Creative Commons Attribution Share-Alike 3.0 United States License.

CreativeCommons.org scraping mechanism for Attribution

This is slick. If you click through to creativecommons.org from any of the image or text links above, you reach a page that offers HTML for use when attributing the work in other web content.

Here is that HTML, rendered:

My Life as a Dog (Phillip Farber) / CC BY-SA 3.0

and here as raw HTML: 

<div xmlns:cc="http://creativecommons.org/ns#" 
     xmlns:dct="http://purl.org/dc/terms/" 
     about="http://babel.hathitrust.org/common/mdp/sample-RDFa.html">
  <span property="dct:title">My Life as a Dog</span> 
  (<a rel="cc:attributionURL" property="cc:attributionName" 
      href="http://babel.hathitrust.org/common/mdp/sample-RDFa.html">Phillip Farber</a>) 
   / 
   <a rel="license" 
      href="http://creativecommons.org/licenses/by-sa/3.0/us/">CC BY-SA 3.0</a>
</div>

This HTML was generated by CC scraping the cc:attributionName and cc:attributionURL attributes on the hidden span that wraps the author's name (Phillip Farber). CC will not generate this sample HTML if either of these attributes are missing.

You may notice a CC icon at the far right end of the Firefox address line. Click on that to see this same attribution information.

Here's the raw HTML of that hidden span that lives in our (not CC's) HTML: 

<span style="display:none" 
      href="http://babel.hathitrust.org/common/mdp/sample-RDFa.html" 
      property="cc:attributionName" 
      rel="cc:attributionURL">Phillip Farber
</span>
The attribution URL is to the page itself. For purposes of attribution, if the author had a URL to his or her work, that would be specified but, in our case, I'm just re-iterating the page itself.

A Note on the use of FOAF in these copyright statements

The web page containing RDFa metadata is the default Subject in the Subject-Predicate-Object RDF triple. This is a bit weird in pageturner. The pageturner page is not the Subject of the CC license, the work displayed within the page is. Here the FOAF ontology is helpful.

I'm using the foaf:isPrimaryTopicOf relation to say the author is licensing the "primary topic of the pageturner page", namely the work it displays. This appears to be a W3C recommended Best Practice. The following two snippets of HTML implement this relation. 

<link about="[_:MyLifeAsADog]" 
      rel="foaf:isPrimaryTopicOf"
      href="http://babel.hathitrust.org/common/mdp/sample-RDFa.html"/> 
<span about="[_:MyLifeAsADog]" 
      property="dc:creator">Phillip Farber</span>

The strange syntax "[_:MyLifeAsADog]" is RDF for a "blank node". What is says is "There is something (which we shall call 'MyLifeAsADog') which is the primary topic (presumably named by the dc:title element) of the page at http://babel.hathitrust.org/common/mdp/sample-RDFa.html." We can now refer to this node as the creation that Phillip Farber is licensing.
