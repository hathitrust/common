The Unicorn Design is a set of stylesheets, fonts, and Javascript to define the common elements across HathiTrust properties. These are maintained in the Git repo /htapps/repos/common.git, under the path common/web/unicorn.

When deployed, these elements can be linked with URLs like: //babel.hathitrust.org/common/unicorn/css/common.css (protocol-less to simplify http vs. https handling).

Does anyone think linking to the common stylesheet from babel.hathitrust.org introduce any issues?

Developing with Sass

The stylesheets are being managed with the a combination of Sass, Compass, and Susy. Sass sets up a number of conveniences that make it easier to develop/extend complex designs and then generating compact, production-ready CSS.

Developing with CSS

In environments were its simpler to modify CSS, developers can link to the generated CSS and simply add/extend classes as necessary.

Icon Fonts

Small graphic files have been replaced with using glyphs from a custom icon font. Icon fonts have a number of advantages over graphic files chiefly in how they can be styled and respond to the user scaling the browser text.

Javascript

A small number of common Javascript widgets are provided by the Unicorn Design. Most have been borrowed from Bootstrap, with HathiTrust styling.

Grid/Layout

The core Unicorn style will define a grid layout.

Vendors

External resources are being maintained in common/web/unicorn/vendor to keep them isolated from any local development. These are not submodules; most of the external resources are actually generated byproducts, not source.
