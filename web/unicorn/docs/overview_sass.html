Several approaches exist:

  • The simplest option is to use Sass to manage a local stylesheet (e.g. pt.sass), assuming that the application links to the processed common stylesheet (e.g. //babel.hathitrust.org/.../common.css).

  • pt.sass can import common.sass: this makes all the classes/modules defined in the common stylesheet available to rules defined in pt.sass, and all the rules are compiled into a single pt.css. For stylesheet rules this mimics existing HTDEV practices of building a single CSS file.

Example: HathiTrust Catalog

Working with the catalog:

  • common.git is checked out at $SDRROOT/common
  • local Sass work happens in $SDRROOT/web/static/unicorn
    • sass contains the Sass sources
    • css contains the generated CSS files
  • web/static/unicorn/sass/catalog.sass starts with @import "../../../../common/sass/common";
  • A local rule (e.g. styling #bd) can now borrow rules and functions from the common stylesheet: 
        #bd {
        @extend .row;
        @include rem(font-size, 14);
    }

Working with Sass

Set up the working space: compass create web -r susy -u susy --css-dir=css --javascripts=js -s expanded --relative-assets

To compile to CSS, execute compass compile inside the web directory.

You can run compass watch to compile to CSS when the Sass files are updated.
